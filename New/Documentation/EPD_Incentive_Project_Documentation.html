Section 01

Project Overview

Background, business objectives, and stakeholder map

The EPD (Established Pharmaceuticals Division) Incentive Management System is a custom web application designed to replace the existing manual Excel-based incentive calculation workflow used by Abbott's Kazakhstan affiliate.

Currently, quarterly incentive payouts for medical and pharmaceutical representatives are calculated in a complex multi-sheet Excel workbook (EPD New Incentive File.xlsx). This workbook interlinks staff data, product portfolios, sales actuals vs. plans, call frequency targets, and coaching activities to produce final incentive amounts in Local Currency (LC = KZT).

💡 Business Driver The current Excel model requires manual updates each quarter, is error-prone when staff changes occur mid-quarter, lacks audit trails, requires manual consolidation across countries, and cannot produce real-time visual reporting for management review.

Scope & Objectives

🎯
Primary Goal

Automate the quarterly incentive calculation engine to replace the Excel workbook with a structured database-backed web application.

📈
Dashboard Reporting

Provide an interactive management dashboard with dual-view reporting: Actual vs. Plan Performance + Financial Summary.

🔒
Data Integrity

Enforce strict column typing in PostgreSQL, eliminate formula dependencies on external workbooks, and maintain audit history.

🌍
Multi-Country Ready

Support multiple countries (KZ = Kazakhstan, GE = Georgia) and per-quarter exchange rate management for LC/USD conversion.

Incentive Calculation Logic

The complete formula, coefficients, and business rules for computing each rep's payout

Core Formula

// TOTAL INCENTIVE (LC) =
  Incentive Amount (Sales Result)
  + Incentive Amount (TCFA)
  + Incentive Amount (Coaching / TIC)
  + Field Work Achievement Bonus

// WHERE:
Target Base LC = Target Incentive for Quarter LC × Reimbursable Months %
Target Sales Result LC = Target Base LC × 0.80 // 80% weight for sales
Target TCFA LC = Target Base LC × 0.20 // 20% weight for TCFA
Figure 3.1 — EPD Incentive Calculation Component Breakdown

Component 1: Sales Result Incentive

This is the largest component (~80% of total target base). It rewards reps for achieving their sales plans across all product portfolios.

  1. Compute weighted % Achievement per Portfolio For each portfolio (P1–P4): sum (ACT product values / PLAN product values) weighted by portfolio share %
  2. Compute overall % Achievement % Overall = Σ (Portfolio Share × Portfolio % Achievement)
  3. Look up Payment Coefficient Based on overall % Achievement, find the applicable multiplier from the coefficient table (see Section 3.5)
  4. Apply per-Portfolio coefficient (where applicable) Some portfolios may have individual multipliers that apply to that portfolio's incentive contribution
  5. Sum all portfolio incentive contributions Incentive Amount (Sales Result) = Σ [Portfolio_i Target × Coeff_i]
// Portfolio Incentive Contribution:
P_incentive = (Target_Base_LC × Target_Sales_Result_pct) × Portfolio_Share × Payment_Coefficient

// Example (Line 1, P1=CREON+DUPHALAC, Share=50%, Coeff=2.5):
P1_incentive = 316,420 × 0.50 × 2.5 = 395,525 LC

Component 2: TCFA Incentive (Target Call Frequency Achievement)

TCFA measures how well a rep hit their planned number of customer visits/calls during the quarter. Each rep has a personal visit target (CP_Personal_target), tracked per region.

TCFA AchievementTCFA CoefficientIncentive Amount
≥ 95%1.00 (Full)Target TCFA LC × 1.0
90%–94%0.50 (Partial)Target TCFA LC × 0.5
< 90%0.00 (None)No TCFA incentive paid
✅ TCFA Data Source TCFA % values come from CRM system records (managed by region in Kazakhstan). Each territory (e.g., KZ-ALM1, KZ-SKO) has a Regional Manager responsible for their team's TCFA reporting. The TCFA.csv export contains Q1 and Q2 actuals per rep.

Component 3: Time in Coaching / Field Work (Regional Managers Only)

This component applies only to Regional Managers. It rewards the time spent accompanying Medical Representatives in the field (coaching visits).

  • Target: 3 coaching sessions per month × 3 months = 9 sessions per quarter
  • Measurement: Monthly % achievement (Actual sessions / Planned sessions)
  • TIC Coefficient: 1.0 if average monthly TIC ≥ 50%, 0.0 if below
  • Impact: Can add up to 20% additional incentive on top of the base for RMs

Payment Coefficient Table

The payment coefficient is the core multiplier applied to incentive targets. It creates a progressive scale that rewards high performance and penalizes underperformance:

% Achievement Band Coefficient Incentive Multiple Business Interpretation
< 70%0.00NoneSignificantly below plan — no incentive
70% – 79%0.50Half incentiveMaterially below plan
80% – 89%0.70–0.90Scaled partialBelow plan, partial reward
90% – 99%0.80–1.00Near-full (~90%)Near plan — rewarded appropriately
100%1.00Full targetPlan achieved exactly
100% – 109%1.20–1.5020–50% bonusOver-achievement — bonus tier
110% – 119%1.70–2.0070–100% bonusStrong over-achievement
120% – 129%2.15–2.302x+ bonusExceptional performance
≥ 150%2.50 (cap)2.5x capCapped at 2.5× to manage payout risk
⚠️ Per-Portfolio Coefficients The data shows that each portfolio (P1, P2, P3, P4) can receive an independent multiplier, not just an overall one. This means the calculation engine must apply coefficients at the portfolio level, not just the rep level. The "check sales" and individual "Payment coefficient" columns in INPUT_ASSUMPTIONS_CALC confirm this.

Reimbursable Months % — Impact on All Components

The Reimbursable Months % scales the Target Incentive Base before any coefficient is applied. This means partial-quarter employees have proportionally lower targets AND proportionally lower maximums:

Target Base_adjusted = Target Incentive for Quarter LC × Reimbursable Months %

// Example: Rep joined mid-quarter (67% reimbursable):
Target for Quarter = 395,525 LC
Reimbursable = 67% → Target Base = 263,683 LC
// All subsequent incentive calculations use 263,683 as the base

Key SQL Schema Snippet

-- ══════════════════════════════════════
-- EPD Incentive — Core Tables
-- PostgreSQL 15+ | UUID extension required
-- ══════════════════════════════════════

CREATE EXTENSION IF NOT EXISTS "uuid-ossp";

CREATE TABLE representative (
  rep_id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  employee_code    VARCHAR(20) UNIQUE NOT NULL,
  full_name        VARCHAR(150) NOT NULL,
  position         VARCHAR(60) NOT NULL,  -- 'Medical Representative' | 'Regional Manager' | 'Pharm Representative'
  country          CHAR(2) NOT NULL,      -- 'KZ' | 'GE'
  region           VARCHAR(50),
  start_date       DATE NOT NULL,
  is_active        BOOLEAN DEFAULT TRUE,
  created_at       TIMESTAMPTZ DEFAULT NOW()
);

CREATE TABLE quarter (
  quarter_id        UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  year              SMALLINT NOT NULL,
  quarter_number    SMALLINT NOT NULL CHECK (quarter_number BETWEEN 1 AND 4),
  country           CHAR(2) NOT NULL,
  exchange_rate_lc_usd DECIMAL(12,4),
  start_date        DATE NOT NULL,
  end_date          DATE NOT NULL,
  status            VARCHAR(20) DEFAULT 'active', -- 'active' | 'locked' | 'archived'
  UNIQUE (year, quarter_number, country)
);

CREATE TABLE incentive_result (
  result_id           UUID PRIMARY KEY DEFAULT uuid_generate_v4(),
  rep_id              UUID NOT NULL REFERENCES representative(rep_id),
  quarter_id          UUID NOT NULL REFERENCES quarter(quarter_id),
  target_base_lc      DECIMAL(14,2) NOT NULL DEFAULT 0,
  sales_result_lc     DECIMAL(14,2) NOT NULL DEFAULT 0,
  tcfa_incentive_lc   DECIMAL(14,2) NOT NULL DEFAULT 0,
  coaching_lc         DECIMAL(14,2) NOT NULL DEFAULT 0,
  field_work_lc       DECIMAL(14,2) NOT NULL DEFAULT 0,
  total_incentive_lc  DECIMAL(14,2) GENERATED ALWAYS AS
                      (sales_result_lc + tcfa_incentive_lc + coaching_lc + field_work_lc) STORED,
  total_incentive_usd DECIMAL(14,2),
  status              VARCHAR(20) DEFAULT 'draft', -- 'draft' | 'calculated' | 'approved' | 'paid'
  calculated_at       TIMESTAMPTZ,
  approved_by         VARCHAR(100),
  approved_at         TIMESTAMPTZ,
  UNIQUE (rep_id, quarter_id)
);

Section 07

Application Module Specification

Required modules and features for the full production application

Module 1 — Authentication & RBAC

  • JWT-based login with role assignment (Admin, Finance, Manager, Read-only)
  • Password reset, session management, audit log of all logins
  • Country-scoped data isolation (KZ users see only KZ data)

Module 2 — Data Management (Admin)

  • Staff Management: Add / edit / deactivate representatives, record leave and position changes
  • Quarter Setup: Create quarters, set exchange rate LC/USD, lock quarters post-approval
  • Promo Line Configuration: Configure portfolio–product mappings per promo line per quarter
  • Payment Coefficient Table: Update threshold bands and multipliers per business cycle

Module 3 — Data Import Pipeline

  • Sales Actuals Import: Upload CSV or Excel, validate columns, display import preview with error summary before committing
  • TCFA Import: Upload TCFA achievement file per region per quarter
  • TIC Import: Upload Time in Coaching data for RMs
  • Manual Override: Allow Finance to override individual fields with audit reason

Module 4 — Incentive Calculation Engine

  • Run quarterly calculation for all reps or individual re-run per rep
  • Calculation versioning: keep multiple calculation runs, mark one as "official"
  • Produce calculation audit trail: inputs used, coefficient applied, result per component
  • Auto-convert LC to USD using quarter exchange rate

Module 5 — Reporting Dashboard

  • Dual-view (Detailed + Financial Summary) with all 6 filters
  • Summary KPI cards: Total Pool, Avg Incentive, Over/Under achievers count, TCFA avg
  • Team leaderboard (sort by total incentive, TCFA %, achievement %)
  • Export current filtered view to Excel or PDF
  • Quarter-over-Quarter comparison chart per rep or team

Module 6 — Approval & Payout Workflow

  • Finance reviews calculated results → status: Draft → Reviewed
  • Country Manager approves → status: Approved
  • Generate payment file (CSV/Excel) for payroll system integration
  • Mark as Paid → locks record from editing
  • Email notifications to relevant stakeholders at each status change

  4. Is there an active external workbook dependency (BC_KZ_2017_V10_FINAL.XLSM) that needs to be ingested into the database?
